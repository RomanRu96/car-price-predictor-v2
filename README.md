# 🚗 Car Price Predictor

ML-пайплайн для предсказания стоимости автомобилей по пробегу, мощности, году выпуска и марке.  
Проект реализован с фокусом на устойчивость, воспроизводимость и готовность к продакшену.

## 🛠 Tech Stack
- **Язык:** Python 3.13
- **ML:** PyTorch, scikit-learn, pandas, numpy
- **Сериализация:** joblib
- **Оптимизатор:** Adam (lr=0.001)

## 📁 Структура проекта


```text
car_price_predictor/
├── train.py              # Обучение, оценка, сохранение артефактов
├── predict.py            # Инференс: загрузка модели и прогноз для новых данных
├── model.pth             # Веса нейросети
├── preprocessor.joblib   # Пайплайн предобработки (StandardScaler + OneHotEncoder)
├── price_scaler.joblib   # Скейлер для целевой переменной (цены)
├── requirements.txt      # Зависимости
└── README.md             # Документация

```

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt

```

### 2. Обучение модели

```bash
python train.py

```
Сохраняет: `model.pth`, `preprocessor.joblib`, `price_scaler.joblib`

### 3. Предсказание

```bash
python predict.py

```
Загружает артефакты и выводит прогнозы для тестовых конфигураций.

## 📊 Модель и метрики

- **Архитектура:** MLP (1 скрытый слой, 16 нейронов, ReLU)
- **Входы:** 7 признаков (3 числовых + 4 one-hot encoded марки)
- **Целевая переменная:** нормализована через отдельный `StandardScaler`
- **Результаты (на синтетических данных):**
  - `R² Train`: ~99.91%
  - `R² Test`: ~98.71%
  - `Loss`: ~0.001

> ⚠️ Метрики получены на учебном датасете (12 записей). На реальных данных ожидается снижение качества из-за шума и нелинейных зависимостей.

## 💡 Ключевые архитектурные решения

- **ColumnTransformer + имена колонок** — защита от поломки при изменении порядка данных. Никаких хардкод-индексов `[0], [1]`.
- **Раздельные скейлеры для признаков и цены** — предотвращает утечку данных и позволяет корректно денормализовать прогноз в рублях.
- **`handle_unknown="ignore"` в OHE** — модель не падает при встрече новой марки авто в продакшене.
- **Строгое разделение `fit_transform` / `transform`** — исключает data leakage: правила нормализации учатся только на train.
- **Воспроизводимость** — зафиксированы `random_state=42` и `torch.manual_seed(42)`.
